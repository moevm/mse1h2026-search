# Проверка Typesense

## Запуск скриптов

### 1. Запуск Typesense в Docker

Для начала необходимо запустить Typesense. Это можно сделать следующей командой:

```bash
docker run -d \
  -p 8108:8108 \
  -v /tmp/typesense-data:/data \
  typesense/typesense:27.1 \
  --data-dir=/data \
  --api-key=xyz
```

### 2. Импорт данных

Скрипт `setup_typesense.py` выгружает данные из исходной базы MySQL, создает коллекцию в Typesense (если её нет) и индексирует документы.

```bash
python setup_typesense.py
```

> Флаг `--recreate` можно использовать, чтобы удалить старую коллекцию Typesense и создать её заново перед импортом.

### 3. Оценка качества поиска

Скрипт `evaluate_search.py` прогоняет набор запросов из JSON-файла и считает метрики качества поиска (Hit@k, MRR).

```bash
python evaluate_search.py queries.json
```

*(Вместо `queries.json` укажите путь к вашему файлу с запросами).*

## Файл конфигурации (`config.py`)

В `config.py` вынесены все основные настройки проекта, чтобы было удобно менять их в одном месте, не залезая в основной код.

---

## Гибридный поиск

Используется **гибридный поиск** — комбинация текстового (keyword) и семантического (vector) поиска.

- **Модель:** `ts/multilingual-e5-small` — встроенная мультиязычная модель (100 языков, включая русский, 384 измерения).
- **Auto-embedding:** Typesense автоматически генерирует эмбеддинги из полей `pagetitle`, `longtitle`, `description`, `introtext`, `content` при индексации.
- **Rank Fusion:** Результаты keyword- и semantic-поиска объединяются формулой `score = (1 − α) × K + α × S`, где `α = 0.55` (55% семантика, 45% ключевые слова). Параметр `alpha` настраивается в `config.py`.

> При первом запуске Typesense скачает модель (~100 MB). Индексация будет медленнее из-за генерации эмбеддингов. Обязательно используйте `--recreate` для пересоздания коллекции с новой схемой.

## Итоговое решение

По результатам бенчмарка для production-среды выбрана модель `multilingual-e5-small`, показавшая наилучшие общие метрики качества гибридного поиска (`MRR` и `Hit@k`).

Оптимальное значение Alpha = 0.80 было определено на основе расчета средневзвешенного показателя MRR для ключевых языков (RU, EN, DE), которые в совокупности составляют более 92,7% поисковой базы.

Выбор этого значения позволяет достичь максимальной интегральной эффективности системы. При этом мы идем на осознанный компромисс: метрики для английского и немецкого языков немного снижаются по сравнению с их локальными максимумами (например, MRR для EN снижается с 0.459 до 0.447). Однако, поскольку русскоязычные документы составляют подавляющее большинство базы (~83%), именно значение 0.80 обеспечивает наивысшую точность поиска для основного объема данных, минимизируя общую погрешность системы.
