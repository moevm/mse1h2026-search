# Проверка Apache Solr 

## Инструкция по запуску

### 1. Подготовка окружения
Убедитесь, что у вас установлен **Docker Desktop** и **Python 3.10+**.

Установите необходимые библиотеки Python:
```bash
pip install -r requirements.txt
```

### 2. Запуск инфраструктуры
Перейдите в папку решения и запустите контейнеры (Solr и MySQL):
```bash
docker-compose up -d
```

### 3. Импорт дампа базы данных
Положите ваш SQL-дамп (`etu.ru.2025.12.26-processed.sql`) в текущую папку и выполните импорт:
```powershell
cmd /c "docker exec -i mysql_etu mysql -u root -proot etu_db < etu.ru.2025.12.26-processed.sql"
```

### 4. Экспорт данных в TSV (Стабильный мост)
Из-за большого объема данных (поле `content` содержит тяжелую верстку и Base64), прямое подключение Python к MySQL может обрываться. Используйте эту команду для выгрузки данных в локальный файл `data.tsv`:
```powershell
cmd /c "docker exec -i mysql_etu mysql -u root -proot --default-character-set=utf8mb4 -e ""SELECT id, pagetitle, longtitle, description, alias, REPLACE(REPLACE(REPLACE(introtext, '\r', ' '), '\n', ' '), '\t', ' '), REPLACE(REPLACE(REPLACE(content, '\r', ' '), '\n', ' '), '\t', ' '), published, deleted, parent, template FROM modx_site_content WHERE published=1 AND deleted=0 AND searchable=1"" etu_db > data.tsv"
```

### 5. Индексация в Solr
Запустите скрипт, который создаст схему в Solr и загрузит данные из файла:
```bash
python setup_solr.py
```

### 6. Запуск бенчмарка
Для получения метрик качества поиска по всем языкам запустите:
```bash
python evaluate_search.py
```

### Анализ и выводы

### 1. Эффективность ранжирования
Использование парсера eDisMax позволило гибко настроить веса полей (приоритет заголовка страницы установлен х15 относительно контента). Это обеспечило точность первого попадания (Hit@1) на уровне 27.5%, что является хорошим показателем для классического текстового поиска без применения векторных эмбеддингов.
### 2. Сильные стороны решения
*   **Высокая релевантность (MRR 0.3286):** Если поисковая машина находит документ, он с высокой вероятностью оказывается на верхних позициях выдачи.
*   **Универсальная мультиязычность:** Использование анализатора text_general обеспечило стабильный поиск на азиатских (VN, CN) и европейских языках без необходимости тонкой настройки лингвистики под каждый отдельный язык.

### 3. Обнаруженные ограничения
*   **Строгость поиска (No Results 55.8%):** Из-за отсутствия морфологического стемминга в универсальном анализаторе, более половины запросов не возвращают результат (особенно критично для русского языка).
*   **Особенности индексации:** Наличие в БД "тяжелого" контента (Base64-изображения, скрипты) требует предварительной очистки данных, так как Solr имеет лимит на длину индексируемых термов (32 Кб).
