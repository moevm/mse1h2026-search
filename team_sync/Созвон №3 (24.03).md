## Протокол встречи 
1. Обсудить результаты 2-й итерации
    - Проведено тестирование поисковых движков, получены предварительные результаты
    - Реализован прототип с использованием мок-данных
    - Формируется сводная таблица сравнения решений (будет готова 25.03)
2. Определить и согласовать метрики качества поиска
    - Чёткие целевые значения метрик не заданы
    - Зафиксированы ориентиры: 10–20% — низкое качество, 40–60% — приемлемый уровень
    - Принято решение ориентироваться на сравнительный анализ
3. Обсудить методы поиска
    - Используется гибридный подход: полнотекстовый и векторный поиск
    - Рекомендовано протестировать различные модели эмбеддингов:
        - DeepPavlov/rubert-base-cased
        - SpanBERT/spanbert-base-cased
        - cointegrated/rubert-tiny2
        - sergeyzh/LaBSE-ru-turbo
        - paraphrase-multilingual-MiniLM-L12-v2
    - Упомянут бенчмарк Massive Text Embedding Benchmark (MTEB)
    - Приоритет: оптимизация эмбеддингов и параметров поиска
    - Лемматизация — более перспективное направление, использование синонимов — менее приоритетно
    - Fine-tuning решения под бенчмарк не нужно
4. Обсудить и спланировать задачи на 3-ю итерацию
    - Сфокусироваться на наиболее перспективных решениях
    - Продолжить улучшение качества выбранных поисковых решений 
    - Определить наиболее эффективный поисковый движок (закончить к середине 3 итерации)
    - Реализовать прототип поиска на одном выбранном движке, покрывающий около 75% сценариев использования
    - В 4-й итерации возможно внедрение прототипа (не гарантировано)
5. Решить технические вопросы
    - Web-component предпочтительнее iframe
    - Планируется интеграция с MySQL (достаточно одной таблицы для прототипа)
    - Инфраструктура:
        - Нам предоставят тестовую среду в Яндекс Облаке
        - Среда используется для бенчмарков, тестирования и демонстрации прототипа
    - Конфигурация:
        - 2 vCPU, 8 ГБ RAM, 50 ГБ HDD
        - ограничение CPU: 50% (с возможностью увеличения)
    - Требуется мониторинг нагрузки и при необходимости оптимизация конфигурации
6. Назначить дату следующего созвона
    - Назначено 07.04 16:00
